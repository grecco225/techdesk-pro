# Resumen para exposición oral — TechDesk Pro

## 1. Introducción
TechDesk Pro es una aplicación web de soporte técnico que permite crear tickets, gestionar incidencias y recibir ayuda automática mediante inteligencia artificial. El sistema está pensado para que usuarios y administradores puedan trabajar de forma organizada, rápida y segura.

La idea principal de la exposición es mostrar que el proyecto no solo funciona como una página web, sino que además integra backend, base de datos, seguridad, autenticación y IA.

---

## 2. Resumen breve del proyecto
TechDesk Pro combina varias tecnologías para construir una solución completa de soporte:
- Frontend: HTML, CSS y JavaScript.
- Backend: Node.js y Express.
- Base de datos: SQLite.
- Seguridad: sesiones, hashes de contraseñas, protección CSRF y cabeceras HTTP.
- Inteligencia artificial: integración con Gemini para ayudar a generar tickets.

En pocas palabras, la aplicación permite registrar problemas técnicos, clasificarlos, asignarlos y darles seguimiento.

---

## 3. Idea central para explicar en voz alta
Puedes decir algo como:

“TechDesk Pro es una plataforma web de soporte técnico que ayuda a los usuarios a reportar problemas de forma sencilla, mientras que el sistema organiza la información, la protege y ofrece ayuda automatizada mediante inteligencia artificial.”

## 4. Guía de exposición oral
### 4.1 Estructura recomendada
Puedes exponer en este orden:
1. Presentación del problema.
2. Qué soluciona la aplicación.
3. Qué tecnologías usa.
4. Cómo funciona la autenticación.
5. Cómo se almacenan los datos.
6. Cómo funciona la IA.
7. Qué medidas de seguridad tiene.
8. Cómo estaría lista para producción.

### 4.2 Tiempo sugerido
- Introducción: 1 minuto.
- Explicación del proyecto: 2 minutos.
- Tecnologías y arquitectura: 2 minutos.
- Seguridad e IA: 2 minutos.
- Cierre: 1 minuto.

### 4.3 Puntos clave que no debes olvidar
- El proyecto es full stack.
- Usa SQLite como base de datos ligera.
- El usuario puede crear tickets.
- La IA ayuda a autocompletar información.
- La app tiene seguridad básica pero importante.
- Está preparada para despliegue en un entorno como Render.

## 6. Simulación de preguntas y respuestas
### Pregunta 1: ¿Qué es TechDesk Pro?
Respuesta: Es una aplicación web de soporte técnico para crear y gestionar tickets de incidencia.

### Pregunta 2: ¿Qué tecnologías usa?
Respuesta: Usa Node.js, Express, SQLite, HTML, CSS, JavaScript y la API de Gemini.

### Pregunta 3: ¿Por qué usar SQLite?
Respuesta: Porque es una base de datos ligera, simple y adecuada para proyectos pequeños o medianos.

### Pregunta 4: ¿Qué hace la inteligencia artificial en el proyecto?
Respuesta: Ayuda a generar sugerencias para el ticket, como categoría, prioridad o solución.

### Pregunta 5: ¿Cómo protege la aplicación los datos del usuario?
Respuesta: Con autenticación, sesiones, hashes de contraseñas, protección CSRF y cabeceras de seguridad.

### Pregunta 6: ¿Qué ventaja tiene esta aplicación para una empresa?
Respuesta: Permite organizar problemas técnicos de forma más rápida, centralizada y eficiente.

### Pregunta 7: ¿Qué pasa si un usuario intenta entrar sin iniciar sesión?
Respuesta: El sistema lo bloquea y lo redirige o le niega el acceso según la ruta protegida.

### Pregunta 8: ¿Por qué es importante el despliegue?
Respuesta: Porque una vez terminada la app, debe poder ejecutarse en un servidor real para que los usuarios la usen.

### Pregunta 9: ¿Qué diferencia hay entre frontend y backend?
Respuesta: El frontend muestra la interfaz al usuario, mientras que el backend procesa datos, valida acceso y gestiona la lógica del sistema.

### Pregunta 10: ¿Cuál es el objetivo principal del proyecto?
Respuesta: Facilitar el soporte técnico mediante una plataforma digital organizada, segura y con ayuda automática de IA.

## 7. Consejos para hablar bien en la exposición
- Habla con seguridad y claridad.
- No leas todo el texto; explica con tus propias palabras.

- Usa ejemplos sencillos, como “un usuario reporta un problema y el sistema lo registra como ticket”.
- Termina con una conclusión fuerte.
